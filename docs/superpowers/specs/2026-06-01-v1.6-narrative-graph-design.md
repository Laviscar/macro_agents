# Macro Agents v1.6 — 叙事驱动图 (Narrative Driver Graph) 设计

> 状态:已与用户敲定全部设计方向(2026-06-01)。本 spec 是实现 plan 的真理来源。

## 一句话目标
把"单一主线 + 累加强度"的旧叙事模型,重构成一张**资产/驱动的有向图(世界树)**:节点是资产与因子,边是"谁驱动谁",证据挂在边上、带时间衰减;系统据此**检测驱动切换并预警**。

## 为什么(动机)
v1.1–v1.5 的叙事建模有三个病根(批 A 只修了 UI 症状,没碰病根):
1. **强度饱和**:strength 是累加 delta,98 次提交后顶到 1.0。
2. **0 证据分支**:LLM 判断开分支,但规则归因算出 0 证据 —— 两套逻辑各说各话。
3. **主线僵化**:单一 `main_default`,标题/角度固定("利率与美元主导"),用户反馈"大家都知道、没参考价值"。

v1.6 用图模型一次性解决:强度=证据加权衰减(不再累加)、证据强制挂边(不存在无证据的边)、多资产多驱动(每个资产是一条独立叙事,角度天然丰富)。

## 一、核心概念

### 1.1 两种节点
- **资产节点 (asset)**:有价格、可盯盘/交易的标的。共 41 个,分三层(见 §2)。字段含 `ticker`。
- **因子节点 (factor)**:驱动价格但本身不可交易的"力量/逻辑"。来自**受控驱动词表**(§3),~16 个。

> 受控词表 = 因子节点 = 边的 `driver_label`,一物三用。

### 1.2 边 (edge) = "谁驱动谁"
有向边 `src → dst`(dst 恒为资产节点;src 可为因子或资产):
- `sign`:`+1` 推升 / `-1` 压制。**结构属性**(慢变,人/审批把关)。
- `weight`:0–1,这条驱动**当前**多强。**动态属性**(快变,叙事管理员自动算)。
- `driver_label`:这条边代表的驱动(来自词表)。
- `supporting_evidence`:挂在这条边上的证据 id + 时间戳。

### 1.3 主导驱动 & 驱动切换(核心价值)
- 一个资产的所有**入边** = 它的候选驱动(对应旧模型的"竞争分支")。
- `dominant_driver` = 当前**权重最大的入边**。
- **驱动切换** = 某资产最强入边的**身份发生变化**(黄金最强入边从 `实际利率` 变成 `央行购金`)→ 触发分歧预警。

### 1.4 两层强度
- **边强度 (edge.weight)**:由挂在该边上的证据加权、按 14 天半衰期衰减后归一化得到。
- **节点强度 (node.strength)**:由该资产**入边的 `sign×weight` 汇总**得到的方向性信念(0=强烈看空该叙事方向,1=强烈看多),`confidence` 同理由证据置信度汇总。

### 1.5 标签维度(B 方案:一主两标签)
每个资产节点带两个标签维度,用于筛选/透视(不另开主线):
- `tags.countries`:LLM 按该节点证据动态打(可多个)。
- `tags.regime`:受控小词表 `{再通胀, 软着陆, 衰退, 滞胀, risk-on, risk-off}`,LLM 每轮归一个。

## 五、数据结构

### 5.1 schemas/graph_node.py
```python
class GraphNode(BaseModel):
    id: str
    kind: Literal["asset", "factor"]
    name: str
    layer: Literal["anchor", "asset_class", "theme", "factor"]
    ticker: str | None = None          # asset 才有
    resident: bool                      # 常驻(L1+L2) vs 事件驱动(L3)
    status: Literal["active", "dormant"] = "active"
    tags_countries: list[str] = []      # asset 才有意义
    tags_regime: str | None = None
    strength: float = 0.5               # 节点方向性信念 0–1
    confidence: float = 0.5
    dominant_driver: str | None = None  # 当前最强入边的 driver_label
    title: str = ""
    thesis: str = ""
    read_line: str = ""
    updated_at: str = ""
```

### 5.2 schemas/graph_edge.py
```python
class EdgeEvidenceRef(BaseModel):
    evidence_id: str
    created_at: str
    contribution: float                 # sign 对齐后的 strength*confidence

class GraphEdge(BaseModel):
    id: str                             # f"{src}->{dst}" 规范化
    src: str                            # 因子或资产 node id
    dst: str                            # 资产 node id
    sign: Literal[1, -1]
    driver_label: str                   # 词表内
    weight: float = 0.0                 # 0–1,时变
    supporting_evidence: list[EdgeEvidenceRef] = []
    status: Literal["active", "candidate"] = "active"
    created_at: str = ""
    updated_at: str = ""
```

## 六、引擎(纯函数,可单测)

### 6.1 graph/strength.py — 边权重 & 节点强度
- `decayed_contribution(ref, now, half_life_days=14) -> float`:`contribution * 0.5**(age_days/half_life)`。
- `compute_edge_weight(edge, now) -> float`:对 `supporting_evidence` 做衰减求和 → `1 - exp(-Σ)`(squash 到 0–1,只增不减彻底废弃)。
- `compute_node_strength(incoming_edges) -> (strength, confidence)`:`strength = sigmoid(Σ sign*weight)` 映射到 0–1(0.5=中性);`confidence = 归一化(证据量/置信度)`。

### 6.2 graph/driver_shift.py — 驱动切换检测
- `dominant_edge(incoming_edges) -> edge | None`:argmax(weight),需 weight ≥ `MIN_DOMINANT_WEIGHT`(默认 0.15)。
- `detect_shift(prev_dominant, new_dominant) -> DriverShift | None`:身份变化时产出事件 `{node, from_driver, to_driver, at}`。
- `contested(incoming_edges, gap=0.1) -> bool`:第二强逼近最强(差 < gap)→ "逼近切换"预警。

### 6.3 graph/repository.py — GraphRepository
- 首次运行:从 `config/*.yaml` 种子化 nodes + edges(幂等)。
- CRUD nodes/edges;`incoming_edges(node_id)`;`assets_with_new_evidence(since)`。
- `promote_candidate_edge(edge_id)` / `reject_candidate_edge(edge_id)`。

## 七、叙事管理员重构(consolidation 阶段)

每轮 consolidation(watermark 增量),对**收到新证据的资产节点**:
1. **归因**(LLM,受控映射):把每条新证据映射到该资产的某条**已有入边**(给出 `driver_label` ∈ 词表 + sign 对齐)。映射不到任何已有边 → 走第 2 步。
2. **提名新边**(LLM,GraphRAG 式三元组):抽 `(因子/资产, 驱动关系, 资产)`,若 `driver_label` 在词表内且边不存在 → 写 `candidate_edges/` 等用户确认(**不自动进图**)。
3. **硬约束(B2)**:没有 ≥1 条归因证据,不新建节点/边;LLM 开"分支"必须同时交出触发证据 id。
4. **重算**:`compute_edge_weight` → `compute_node_strength`。
5. **检测切换**:`detect_shift` / `contested` → 写 `driver_shifts/`(= 分歧预警)。
6. **标签 & 文案**(LLM):更新 `tags_regime`、`tags_countries`、`title`、`thesis`、`read_line`。
7. **L3 休眠**:连续 21 天无新证据的 L3 节点 `status=dormant`(状态保留)。

LLM 用 narrative-tier client(已有三层 LLM 配置)。审计席位(V1.5 AuditPanel)继续可选介入"切换判断"。
