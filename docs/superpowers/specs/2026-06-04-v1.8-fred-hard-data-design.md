# Macro Agents v1.8 — FRED 硬宏观数据接入 (Hard Macro Data Layer) 设计

> 状态:已与用户 brainstorm 敲定(2026-06-04)。分支 `v1.8-dev`(从 main 拉)。

## 一句话目标
把 FRED(圣路易斯联储)的硬宏观时间序列(美债收益率/实际利率/通胀预期/政策利率/利差/美元/VIX/失业/CPI…)作为**数值真相层**接入:拉取真值存独立读数库,**注入委员会上下文 + 关联到对应因子/锚节点**,让委员引真数不编数、补驱动图核心因子的硬数据缺口。

## 设计原则
- **数值真相层,不走 LLM、不改图**:FRED 真值不经 LLM 改写;**不修改边权重/节点 strength**(图仍由证据驱动)。FRED 是叠加的"硬读数"层。
- **动机**:v1.7 黄金圆桌暴露 LLM 编造 "实际利率 1.93%" 这类具体数字;FRED 提供 DFII10 等真值,委员(尤其数据派)可引真值。
- **UI 延后**:按用户工作流(先功能后 UI,见 [[defer-ui-until-content-review]]),**本版只做数据/逻辑层**;读数面板/节点显示读数等 UI 留到用户内容审核后再做。

## 非目标
- 不把 FRED 当新闻/证据源(不走 triage/analyze/consolidate)。
- 不让 FRED 改 strength/边权重。
- 不做 UI(本版)。

## 二、适配器 `sources/fred.py::FredClient`

```python
class FredReading(BaseModel):
    series_id: str
    label: str
    unit: str
    node_id: str | None
    value: float
    date: str            # 最新观测日期
    prev: float | None
    change: float | None  # value - prev
    fetched_at: str

class FredClient:
    def __init__(self, api_key=None, api_key_env="FRED_API_KEY", fetcher=None): ...
    def fetch_observation(self, series_id) -> tuple[float, str, float | None]:
        # GET /fred/series/observations?series_id=X&api_key=K&file_type=json&sort_order=desc&limit=2
        # 返回 (最新值, 最新日期, 前值);FRED 缺测值为 "." -> 跳过取下一个有效值
```
- 端点 `https://api.stlouisfed.org/fred/series/observations`;`fetcher` 可注入(测试)。
- 解析:取 `observations` 倒序前两个**有效**值(跳过 `"."`);`change = 最新 - 前值`。
- 失败(网络/无 key/无数据)→ 抛 `FredError`,由 stage 捕获跳过。

## 三、存储 `repositories/fred_repository.py::FredRepository`
- `storage/fred_readings/<series_id>.json` 存 `FredReading`。
- `save_reading(reading)` / `list_readings() -> list[FredReading]`。
- `reading_for_node(node_id) -> FredReading | None`(取该节点对应的硬读数)。
- `general_readings() -> list[FredReading]`(node_id 为空的)。

## 四、拉取 stage `pipelines/stages.py::fetch_fred_readings`
```python
def fetch_fred_readings(fred_repo, fred_client, series_config) -> dict:
    ok = errors = 0
    for s in series_config:                       # series_config: list[dict] from yaml
        try:
            value, date, prev = fred_client.fetch_observation(s["series_id"])
            fred_repo.save_reading(FredReading(..., change=(value-prev) if prev is not None else None, fetched_at=now_iso()))
            ok += 1
        except FredError:
            errors += 1                            # skip + warn
    return {"fred_ok": ok, "fred_errors": errors}
```
- **低频、0 LLM**。`run_loop` 加一个 stage,周期 `FRED_REFRESH_SECONDS`(默认 21600 = 6h)。
- 在 `build_run_loop` 构造 `FredRepository` + `FredClient`(读 `FRED_API_KEY`)+ 读 `config/fred_series.yaml`;无 key 时 stage 跳过(告警,不崩)。

## 五、委员会上下文注入(解决"编数字")
- 在召开圆桌组装 context 时(v1.7 的 `_committee_context` / convene),追加:

- 注:committee 模块通过依赖注入拿 `fred_repo`(UI/脚本召开时传入);无 FRED 数据时该块省略。
