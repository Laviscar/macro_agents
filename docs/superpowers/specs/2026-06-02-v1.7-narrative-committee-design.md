# Macro Agents v1.7 — 叙事委员会圆桌 (Narrative Committee Roundtable) 设计

> 状态:已与用户 brainstorm 敲定(2026-06-02)。本 spec 是实现 plan 的真理来源。
> 分支 `v1.7-dev`(从 main 拉)。

## 一句话目标
在驱动图出现**逼近切换**的关键时刻,由用户**人工召开**一个可配置(人格/专长/LLM/skill)的"叙事委员会"做**多轮圆桌讨论**,主席综合出结论(切换是否将至、对多空的影响、关键分歧)。结论作为**咨询层**叠加在世界树上——**标注、展示,但不改图**。

## 设计原则
- **人在环 + 控成本**:跨阈值只"标记待召开",**人工点「召开」才跑 LLM**;席位数/轮次/成员随时可调;自动组阵是规则化的(不额外烧 LLM);召开前显示预估调用次数。
- **咨询不动图**:委员会结论**不修改边权重**(图仍纯证据驱动、可审计);只打徽章、影响 UI 显示/排序。
- **复用 V1.5**:圆桌讨论复用 `agents/audit.py::AuditPanel`(cross/p2p 多轮),席位扩展为带人格/专长/skill;新增"主席综合"。
- **描述性 skill**:v1.7 的 skill 是**提示词层面的视角**(塑造委员盯什么),**不执行工具**;可执行(接 V1.3 `ToolRuntime`)留 v1.8(schema 预留)。

## 非目标(YAGNI / v1.8+)
- 不让委员会改图权重 / 不自动执行交易或建议。
- 不做可执行 skill(工具调用)——schema 预留,逻辑留 v1.8。
- 不自动召开(始终人工门控)。

---

## 一、触发机制(去抖,人工门控)

- **逼近度** `ratio = 次强入边权重 / 最强入边权重` ∈ [0,1]。
- **阈值档** `0.75 / 0.85 / 0.90`(可配 `COMMITTEE_TRIGGER_LEVELS`)。
- 某资产 ratio **向上跨过**某档 → 生成 `PendingConvocation(asset, level, ...)`,**每档只生成一次**。
- **滞回去抖**:记录每个资产已触发的最高档;ratio 必须**跌回该档下方再升上来**才会对该档重新触发。状态存 `storage/committee_trigger_state.json`(`{asset_id: highest_level_fired}`)。
- 触发计算挂在 **consolidation 末尾**(已有 `pending`/`contested` 数据),写入 `storage/committee_pending/<asset>.json`。
- **只标记,不跑 LLM。** 召开是人工动作。

## 二、数据模型(schemas/)

### 2.1 CommitteeSkill(skill 库条目)
```python
class CommitteeSkill(BaseModel):
    id: str
    name: str
    description: str           # 塑造提示词:这位委员从这个角度审视
    tools: list[str] = []      # 预留:v1.8 接 ToolRuntime;v1.7 恒空
```

### 2.2 CommitteeSeat(席位)
```python
class CommitteeSeat(BaseModel):
    name: str
    persona: str               # 受控词表:鹰派/鸽派/逆向/结构派/数据派
    expertise: list[str] = []  # 自由文本/标签:盯哪块(利率/地缘/流动性…)
    llm_tier: str = "auditor_1" # 复用三层+auditor_* 配置;各席位独立 key
    skills: list[str] = []     # 引用 CommitteeSkill.id
```

### 2.3 CommitteeTemplate(预设阵容)
```python
class CommitteeTemplate(BaseModel):
    id: str
    name: str                  # 如"鹰鸽对冲"
    seats: list[CommitteeSeat]
    rounds: int = 1
    mode: Literal["cross", "p2p"] = "cross"
```

### 2.4 PendingConvocation(待召开)
```python
class PendingConvocation(BaseModel):
    asset_id: str
    asset_name: str
    level: float               # 触发档 0.75/0.85/0.90
    leader: str                # 领先驱动
    runner_up: str             # 逼近驱动(挑战者)
    is_reversal: bool
    created_at: str
```

### 2.5 CommitteeVerdict / CommitteeSession(结论 + 纪要)
```python
class SeatRemark(BaseModel):
    seat_name: str
    persona: str
    round: int
    critique: str

class CommitteeVerdict(BaseModel):
    switch_likelihood: Literal["将至", "不确定", "噪音"]
    confidence: float          # 0–1
    positioning: Literal["偏多", "偏空", "中性"]
    positioning_reason: str
    key_disagreements: list[str]   # legend-talk:点出席位间分歧
    summary: str

class CommitteeSession(BaseModel):
    id: str                    # f"{asset}_{at}"
    asset_id: str
    asset_name: str
    level: float
    seats: list[CommitteeSeat]
    rounds: int
    mode: str
    remarks: list[SeatRemark]  # 各席位多轮发言
    verdict: CommitteeVerdict
    created_at: str
```

存储:`storage/committee_sessions/<id>.json`;徽章状态见 §五。

## 四、委员会引擎

### 4.1 committee/trigger.py(纯函数,去抖)
- `crossed_levels(ratio, prev_highest, levels) -> (new_pending_levels, new_highest)`:返回这次新跨过的档 + 更新后的最高档;滞回逻辑在此。
- `evaluate_assets(graph_repo, trigger_state, levels) -> list[PendingConvocation]`:对每个 contested 资产算 ratio、比对 state、产出待召开。consolidation 调用它,持久化 pending + state。

### 4.2 自动组阵(规则化,不烧 LLM)— 5
- `auto_staff(asset, incoming_edges, skill_library, persona_vocab, max_seats) -> list[CommitteeSeat]`:
  - 按该资产**竞争驱动的 driver_label + nature** 匹配 skill(如驱动含"实际利率"→选带 `rates_curve` 的视角;含"避险地缘"→`geopolitics`)。
  - 为"有效张力"分配**对立人格**(至少一鹰一鸽 + 一逆向 + 一求证),最多 `max_seats`。
  - 纯规则,**0 次 LLM**。用户可在召开前手动增删席位/换人格/换 LLM(人在环)。

### 4.3 agents/committee.py(召开,复用 AuditPanel)
- `NarrativeCommittee(seat_clients_with_meta, rounds, mode, chair_client)`:
  - 复用 `AuditPanel` 的 cross/p2p 多轮机制;每个席位的 system prompt 由 **persona + expertise + 勾选 skills 的 description** 拼成。
  - 各轮各席位产出 `SeatRemark`。
  - **主席综合**(chair_client = narrative tier):读全部发言 → 产出 `CommitteeVerdict`(含 `key_disagreements`)。
  - `convene(pending, context) -> CommitteeSession`。
  - LLM 失败:席位失败跳过+告警(沿用 AuditPanel);主席失败 → 退化为"无统一结论"并标注。

### 4.4 上下文组装
给委员会的 context:资产当前主导/挑战驱动 + 两条竞争边的证据 claims + 当前多空(strength)+ 驱动 nature + 近期相关 driver_shifts。

## 五、作用:咨询 + 标注(不动图)

- `convene` 产出 `CommitteeSession` → 存 `committee_sessions/`。
- 同时写一个**轻量徽章** `storage/committee_badges/<asset>.json`:`{asset, switch_likelihood, positioning, session_id, at}`。
- 徽章在 **分歧预警 / 今日叙事** 显示(如"🏛 委员会:切换将至·偏空"),并可让今日排序对"委员会判将至"的资产**加权**(只影响排序,不碰 strength/边权重)。
- **断言级约束**:committee 模块**不调用任何 graph_repo 的 save_edge/save_node 权重写入**;测试保证图状态在召开前后不变。

## 六、UI:独立「叙事委员会」页(第 6 个 tab)

`presenters/committee_presenter.py` + `view_models/committee_view.py`。三区:

1. **配置区**(人工控制):
   - 席位编辑器:增删席位、选 persona(下拉)、expertise(标签)、LLM(tier 下拉)、勾选 skills(来自库)。
   - rounds / mode 选择。
   - 预设模板下拉(选一套一键填充席位)。skill 库一览(只读展示)。
   - 「保存配置」写回 `config/committee.yaml`。
2. **待召开区**:
   - 列 `committee_pending/`(资产 + 逼近档位 + 领先/挑战驱动 + 是否反转)。
   - 每条:可「🤖 自动组阵」(规则填充,可再手改)或用当前配置;显示**预估 LLM 调用次数 ≈ 席位×轮次 + 1(主席)**;「召开圆桌」按钮(点了才跑)。
3. **历史区**:
   - 列过往 `committee_sessions/`:主席结论(switch_likelihood/positioning/key_disagreements/summary)+ 展开看各席位多轮发言。

分歧预警页加入口:「🏛 N 个待委员会」+ 已有徽章显示在对应资产条目。

## 七、Token 控制(贯穿)
- 人工门控召开(主闸)。
- 自动组阵规则化(0 LLM)。
- 预设模板(0 LLM)。
- 召开前显示预估调用次数。
- rounds 默认 1、`COMMITTEE_MAX_SEATS=5` 上限。
- 席位用 `auditor_*` 独立 key/tier,可指便宜模型。

## 八、复用与集成
- `agents/audit.py::AuditPanel`:cross/p2p 多轮——委员会在其上加 persona/skill 提示 + 主席综合。
- `GraphNarrativeManager.audit_panel` 这个 v1.6 预留 hook **本版不强绑**:委员会是 UI 人工触发的独立流程,不挂进自动 consolidation(避免自动烧钱);hook 可留作未来"自动咨询"。
- 触发计算复用 v1.6 的 contested / incoming_edges。

## 九、测试 / 边界
- 触发去抖:跨档一次、滞回重置、多档递进(纯函数单测)。
- 自动组阵:按驱动匹配 skill、对立人格、≤max_seats(纯函数)。
- 召开:FakeLLMClient,断言 persona/skill 进了提示词、多轮、主席产出 verdict(含 key_disagreements);席位失败跳过;主席失败退化。
- **不动图**:召开前后 graph_nodes/graph_edges 不变(断言)。
- presenter:pending/history/badge view 构建;config 写回往返。
- 配置校验:committee_skills/committee.yaml 结构、persona ∈ 词表、skills 引用存在。

## 十、范围边界(v1.8+)
- 可执行 skill(接 ToolRuntime)。
- 自动召开(去抖直接开会)。
- 委员会"有牙"(改图)——若将来要,需单独设计审计轨迹。
