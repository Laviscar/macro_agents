# v1.8 FRED 硬数据接入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans (inline). Steps use checkbox (`- [ ]`).

**Goal:** 把 FRED 硬宏观时间序列作为数值真相层接入:拉取真值存读数库,注入委员会上下文 + 关联因子/锚节点,让委员引真数不编数。

**Architecture:** FredClient(取观测,跳缺测,算 change)→ FredRepository(storage/fred_readings/)→ fetch_fred_readings stage(低频, 0 LLM, run_loop)→ committee 召开时注入 FRED 真值文本。不走 LLM、不改图 strength/边权重。**本 plan 不含 UI**(留到用户内容审核后)。

**Tech Stack:** Python 3.12, pydantic v2, pytest, PyYAML, stdlib urllib;复用 v1.7 committee/repositories 模式。

参考 spec:`docs/superpowers/specs/2026-06-04-v1.8-fred-hard-data-design.md`(§)。

### Task 3: FredClient.fetch_observation
**Files:** Create `sources/fred.py`; Test `tests/test_fred_client.py`

```python
import json
from sources.fred import FredClient, FredError

def _resp(obs):
    return json.dumps({"observations": obs})

def test_latest_and_prev_and_change():
    fc = FredClient(api_key="k", fetcher=lambda url: _resp([
        {"date": "2026-06-03", "value": "1.93"}, {"date": "2026-06-02", "value": "1.89"}]))
    value, date, prev = fc.fetch_observation("DFII10")
    assert value == 1.93 and date == "2026-06-03" and prev == 1.89

def test_skips_missing_dots():
    fc = FredClient(api_key="k", fetcher=lambda url: _resp([
        {"date": "2026-06-03", "value": "."}, {"date": "2026-06-02", "value": "1.90"},
        {"date": "2026-06-01", "value": "1.85"}]))
    value, date, prev = fc.fetch_observation("DFII10")
    assert value == 1.90 and date == "2026-06-02" and prev == 1.85   # first valid + next valid

def test_no_valid_raises():
    fc = FredClient(api_key="k", fetcher=lambda url: _resp([{"date": "d", "value": "."}]))
    try:
        fc.fetch_observation("X"); assert False
    except FredError:
        pass

def test_missing_key_raises():
    try:
        FredClient(api_key=None, api_key_env="NOPE_FRED").fetch_observation("X"); assert False
    except FredError:
        pass
```
