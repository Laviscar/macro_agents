# V1.3 Harness Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Fix four correctness/reliability gaps flagged in the harness review memo so the eval harness and audit trail become trustworthy: (#3) sessions don't persist narrative result, (#4) eval window drops same-day sessions, (#5) ALLOW policy decisions aren't audited, (#6) session store sqlite reliability.

**Architecture:** Surgical fixes to existing harness modules; no new modules. Each fix is independently testable via TDD.

**Tech Stack:** Python 3.11+, SQLite, existing harness modules. No new dependencies.

---

## Verified current behavior (anchors)

- `harness/loop.py` run(): on success calls `self.session_store.complete_session(session_id, {"stop_reason": ctx.stop_reason})` — `ctx.narrative_result` is discarded.
- `ctx.narrative_result` is set in the UPDATE_NARRATIVE handler to the `update_narrative` tool output `{"main_narrative_id", "branches_count", "commits_count"}`, or stays `None` when narrative update is skipped.
- `harness/metrics.py` reads `main_narrative_id` from each session's `result_json` for narrative_stability → currently always missing → stability degenerates.
- `harness/eval.py` run_eval filters with `window_start <= s["created_at"] <= window_end` (string compare); `created_at` is a full ISO timestamp (e.g. `2026-05-29T14:00:00`), `eval_cli.py` passes date-only bounds (e.g. `2026-05-29`). A same-day session is therefore excluded.
- `harness/runtime.py` execute(): attaches `policy_record` only on DENY / ASK_FOR_APPROVAL; ALLOW results carry no record.
- `harness/loop.py` TOOL_EXEC: emits POLICY_DECISION only `if result.policy_record is not None`; sets `policy_intercepted = result.policy_record is not None`. UPDATE_NARRATIVE handler emits no POLICY_DECISION at all.
- `harness/session_store.py`: single `sqlite3.connect(..., check_same_thread=False)`, no `PRAGMA foreign_keys`, no lock around writes.
- The loop-test fixtures in `tests/test_harness_loop.py` use `ToolRuntime()` **without** a policy engine, so they emit no policy records (safe for #5 changes).

```python
    def execute(self, tool_name: str, input: dict) -> ToolResult:
        if tool_name not in self._tools:
            return ToolResult(tool_name=tool_name, success=False, error=f"Unknown tool: {tool_name}")
        tool = self._tools[tool_name]
        record = None
        if self._policy_engine is not None:
            record = self._policy_engine.record(tool.name, tool.risk_level)
            if record.decision == PolicyDecision.DENY:
                return ToolResult(
                    tool_name=tool_name,
                    success=False,
                    error=f"policy_denied: {record.reason}",
                    policy_record=record,
                )
            if record.decision == PolicyDecision.ASK_FOR_APPROVAL:
                return ToolResult(
                    tool_name=tool_name,
                    success=False,
                    error=f"requires_approval: {record.reason}",
                    policy_record=record,
                )
        try:
            result = self._tools[tool_name].execute(input)
        except Exception as exc:
            return ToolResult(tool_name=tool_name, success=False, error=str(exc), policy_record=record)
        if result.policy_record is None:
            result.policy_record = record
        return result
```

```python
def test_concurrent_event_writes_are_not_lost(tmp_path):
    import threading
    from harness.events import EventType, LoopEvent
    store = HarnessSessionStore(tmp_path / "test.db")
    sess_id = store.create_session()

    def writer():
        for _ in range(20):
            store.record_event(LoopEvent(
                session_id=sess_id,
                event_type=EventType.LOOP_TRANSITION,
                state="PLAN",
                payload={},
            ))

    threads = [threading.Thread(target=writer) for _ in range(5)]
    for t in threads:
        t.start()
    for t in threads:
        t.join()

    events = store.list_events_for_session(sess_id)
    assert len(events) == 100  # 5 threads x 20 events, none lost
```

- [ ] **Step 3: Edit `harness/session_store.py`**

Add `import threading` near the top (after `import sqlite3`).

In `__init__`, after `self._conn.row_factory = sqlite3.Row` and before `self._init_schema()`, add:

```python
        self._conn.execute("PRAGMA foreign_keys = ON")
        self._lock = threading.Lock()
```

Then wrap every write method's `execute(...) + commit()` body in `with self._lock:`. Concretely, guard the bodies of: `create_session`, `complete_session`, `fail_session`, `record_event`, `save_compaction`, `save_eval_run`. For example `record_event` becomes:

```python
    def record_event(self, event: LoopEvent) -> None:
        with self._lock:
            self._conn.execute(
                """
                INSERT INTO harness_events
                    (session_id, event_type, state, payload_json, created_at)
                VALUES (?, ?, ?, ?, ?)
                """,
                (
                    event.session_id,
                    event.event_type,
                    event.state,
                    json.dumps(event.payload, ensure_ascii=False, default=str),
                    event.created_at,
                ),
            )
            self._conn.commit()
```

Apply the same `with self._lock:` wrapping to the other five write methods (keep the existing `rowcount` KeyError checks for `complete_session`/`fail_session` inside the lock). Read methods (`list_events_for_session`, `list_sessions`, `get_compaction`, `list_eval_runs`) do NOT need the lock.
